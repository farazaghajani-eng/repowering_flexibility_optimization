# Contributing to Repowering Flexibility Optimization

Thank you for your interest in contributing to the Repowering Flexibility Optimization project! We welcome contributions from the community and want to make the process as smooth as possible. Please follow the guidelines below to help us maintain the quality of the project.

## Getting Started
1. **Fork the Repository**: Start by forking the repository. This creates a personal copy of the project where you can develop your changes.
2. **Clone the Repository**: Clone your forked repository to your local machine:
   ```bash
   git clone https://github.com/<your-username>/repowering_flexibility_optimization.git
   ```
3. **Create a Branch**: Create a new branch for your feature or bug fix:
   ```bash
   git checkout -b your-feature-name
   ```
4. **Make Changes**: Implement your changes and test them thoroughly.

## Code Style
- Follow the coding style conventions used throughout the project. Ensure that your code is clean and well-organized.
- Use meaningful variable and function names that accurately represent their purpose.
- Ensure that your code includes comments where necessary, especially for complex logic.

## Pull Request Process
1. **Commit Your Changes**: After you've made and tested your changes, commit them to your branch with a clear and concise commit message.
   ```bash
   git add .
   git commit -m "Descriptive commit message"
   ```
2. **Push Your Changes**: Push your changes to your forked repository:
   ```bash
   git push origin your-feature-name
   ```
3. **Open a Pull Request**: Navigate to the original repository on GitHub. Click on the "Pull Requests" tab, then click on "New Pull Request". Select your branch to merge.
4. **Review Process**: Once submitted, your pull request will be reviewed by the team. Be prepared to make additional changes based on feedback.

## Research Standards
- When contributing research-related content, ensure that all data and references are credible and properly cited.
- Follow ethical guidelines in research practices, including proper attribution to sources and respect for confidentiality.
  
Thank you for helping us improve the Repowering Flexibility Optimization project! We look forward to your contributions!
